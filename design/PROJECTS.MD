Projects Page

Purpose

Present selected work in a restrained, editorial format that emphasizes thinking, constraints, and outcomes over visuals or tech stacks.

Scope
	•	Main Projects index page
	•	No heavy animations
	•	No authentication

Page Structure

Header
	•	Title: PROJECTS
	•	Subtext: “Selected work. Some finished, some in progress, all excited to share to y'all. ”

Project List
	•	Vertical list, single column
	•	3–5 projects maximum
	•	Ordered by representativeness, not prestige

Each project entry includes (can create placholder projects for now):
	•	Project name (uppercase)
	•	One-line problem statement
	•	One-line description of what was built / learned
	•	Link: [details]

Example:

PROJECT KIM
Measuring how live-stream moments escape their original platforms.
Built pipelines to compare engagement across short-form platforms.
[details]

Interaction
	•	Clicking [details] links to an external GitHub repository (or similar)
	•	No thumbnails by default
	•	Subtle fade-in on load (same motion language as Home)

Visual Language
	•	White background
	•	Black text
	•	One accent color for links/hover
	•	Same typography system as rest of site
	•	No cards, no shadows, no grids

Footer Note (Optional)

“More experiments coming soon! hopefully”

Future Expansion (Out of Scope)
	•	Filtering or tags
	•	Project timelines
	•	Metrics or testimonials
